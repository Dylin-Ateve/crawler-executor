# 规格驱动开发工作区

本目录按 Spec Kit 风格组织规格驱动开发材料，用于沉淀项目原则、功能规格、技术计划、研究结论、数据模型、契约和任务拆解。

## 工作流程

1. 在 `.specify/memory/constitution.md` 中维护项目原则。
2. 持续修订 `specs/001-scrapy-distributed-crawler/spec.md`，直到需求边界清晰。
3. 逐步完善 `plan.md`、`research.md`、`data-model.md`、`contracts/` 和 `quickstart.md`。
4. 只有在规格和设计材料稳定后，才生成或更新 `tasks.md`。
