# 项目章程：Scrapy 分布式爬虫

## 核心原则

### I. 规格先行

所有实现决策都必须能追溯到 `.specify/memory/`、`state/decisions/` 或 `specs/` 下已记录的约束、需求、假设或澄清结论。

### II. 运维安全优先

爬虫在扩容前必须具备明确的限速策略、Host 级控制、重试边界、黑名单机制和可观测性。

### III. 数据可靠性

HTML 持久化、抓取事实事件投递和下游订阅契约必须在实现前定义失败行为、重放语义和幂等边界。

### IV. 增量交付

系统必须按可独立验证的阶段交付：单节点 PoC、内容持久化与事件 producer、分布式只读消费链路、控制平面接入和自动化部署。

### V. 可度量验收

性能、成本、可靠性和运维目标必须以可度量标准表达，并配套明确的验证步骤。

## 治理规则

- 标记为 `NEEDS CLARIFICATION` 的需求会阻塞详细计划。
- 技术计划必须记录已考虑但未采纳的更简单替代方案。
- 进入实现阶段后，任务必须包含明确的文件路径。
- PoC 验收标准达成前，不进入生产规模扩容。
- 仓库沟通与维护说明默认使用中文；涉及外部合作或多语言文档时再单独标注。
